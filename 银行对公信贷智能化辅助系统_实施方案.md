### 四、数据结构设计

```python
from dataclasses import dataclass
from enum import Enum

class DocumentLevel(str, Enum):
    REQUIRED = "required"      # 必填，不可跳过
    SUGGESTED = "suggested"   # 建议，未上传则警告
    IF_EXISTS = "if_exists"   # 如有则提供

@dataclass
class RequiredDocument:
    code: str                 # "A1", "B2" 等
    name: str                 # 资料名称
    category: str             # "A", "B", "C", "D"
    level: DocumentLevel      # 校验级别
    accepted_types: list[str] # 允许的文件扩展名

# Session State 新增字段
"uploaded_files": dict[str, str]  # {doc_code: filename}
"required_complete": bool         # 必填项是否全部完成
```

## 更新：Human-in-the-Loop + Excel 分流强化（v3.2）

### 更新背景

v3.1 的人工核对环节放在 Phase 1（查看原始 Markdown），存在以下问题：
1. **原始 Markdown 可读性差**：OCR 解析后的表格/数字未结构化，信贷员难以核对
2. **LLM 幻觉风险**：直接用原始 Markdown 生成报告，可能放大 OCR 错误
3. **Excel 分流不够严格**：上传阶段未对 Excel 文件做强制路由

v3.2 将人工核对移至 Phase 2 之后，以结构化表格形式展示财务指标，信贷员可直接修正 AI 提取的数值后再生成报告。

---

### 一、新完整流程

```
Phase1 解析
    ↓
Phase2 财务分析 + 科技特征提取 + 合规筛查
    ↓
★★★ Human-in-the-Loop：核对结构化数据（st.data_editor）★★★
    ↓
Phase3 Gemini 报告生成
```

---

### 二、Human-in-the-Loop 核对内容

#### 核对数据来源

| 数据来源 | 内容 |
|---------|------|
| `phase2_analysis.run_financial_analysis()` | 盈利能力、偿债能力、成长性指标 |
| `phase2_analysis.extract_tech_innovation_metrics()` | 研发费用占比、高新证书、专利数量、团队履历 |
| `phase2_compliance.ComplianceScreener` | 企业/个人维度合规结论、综合风险等级 |
| Markdown 内容解析 | 企业名称、注册资本、成立日期等基本信息 |

#### st.data_editor 展示格式

```python
# 财务指标（Editable）
financial_data = {
    "指标名称": ["毛利率(%)", "净利润(万元)", "资产负债率(%)", "流动比率", "营收CAGR(%)"],
    "AI提取值": [25.6, 1234.5, 58.3, 1.82, 12.4],

```

### 五、Excel 分流保护强化

#### 现有实现（phase1_parser.py，已满足）

- `EXCEL_EXTENSIONS = {".xls", ".xlsx", ".xlsm"}` 分流定义
- 上传后按后缀判断，Excel 文件路由至 `parse_excel_locally()` → pandas

#### 强化措施（app.py 上传阶段）

```python
EXCEL_EXTENSIONS = {".xls", ".xlsx", ".xlsm"}

# 记录每个文件的路由（用于调试/审计）
st.session_state.file_routes = {}
for uploaded in uploaded_files:
    ext = Path(uploaded.name).suffix.lower()
    if ext in EXCEL_EXTENSIONS:
        st.session_state.file_routes[uploaded.name] = "pandas"
        # Excel 文件绝不发 MinerU
    else:
        st.session_state.file_routes[uploaded.name] = "mineru"
```

## 更新：HTML前端模板（v4.0）

### 更新背景

为方便UI定制和前端开发，将原Streamlit界面转换为纯HTML/CSS/JS模板。保留Python后端逻辑不变，前端模板供用户后续修改使用。

### 模板文件

- `bank_credit_assist/templates/index.html` — 完整HTML前端模板

### 技术栈

- 纯HTML5 + CSS3 + Vanilla JavaScript
- 无框架依赖，便于理解和修改
- 响应式设计，支持桌面端

### 功能区域

| 区域 | 功能 |
|------|------|
| 头部 | 系统标题 + 工作流状态标签（7种状态） |
| 资料清单 | A/B/C/D四类18项，支持折叠展开、多选上传、单独删除 |
| 解析操作 | 必填项进度条、文件列表、路由预览(pandas/MinerU)、开始解析按钮 |

| 错误区域 | 错误信息显示、重新开始按钮 |
